Chapter 1. Overview of Docserv²

Docserv² is a tool to publish large-scale documentation websites accommodating multiple products, product versions, and localizations. It focuses on allowing publication of DocBook/AsciiDoc content that is compatible with DAPS. However, it can also accommodate links to other sources of documents.

The Docserv² build server is written in Python/Bash/XSLT and designed to run as a daemon. Its Git repository is at https://github.com/openSUSE/docserv.

Figure 1.1. Docserv² overview


The following elements that are relevant to Docserv²:

  • Servers:

    • Git is used for versioning the sources of the content to publish.

    • The configuration for Docserv² should be stored in a Git repository as well, though a separate one to that of the documentation.

    • The Docserv² build server is the core piece of the publishing process. It builds documents and their associated navigation pages. After building a specific piece of documentation, it copies the documentation output to a backup directory and from there to a publishing server.

    • The build server also provides a REST API that can be used to trigger rebuilds for documentation and check the status of those rebuilds.

    • The publishing server(s) are the servers that users will finally access to get to their documentation. There is no logic that needs to run on the publishing servers. Therefore, any computer running a Web server is suitable as a publishing server.
